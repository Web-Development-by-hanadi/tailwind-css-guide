Introduction what is Tailwind CSS?

Tailwind CSS is a utility-first CSS framework. Instead of writing custom CSS files and inventing class names like .card-title or .btn-primary, you style elements directly in your HTML using small, single-purpose classes — one class per property, such as p-4 for padding, text-blue-600 for color, or flex for layout.

These utility classes are combined directly in the markup to build any design, without leaving the HTML file and without the class-naming decisions that traditional CSS requires. The whole point of this reference page is to make that concrete: every category below shows the classes rendered live, not just described in a table.

Where Tailwind is used

  • Web applications & dashboards — consistent spacing and color without a separate design system to maintain.
  • Landing pages & marketing sites — fast to build and easy to make fully responsive.
  • Rapid prototyping — try a layout directly in the markup before committing to final styles.
  • Component libraries — many modern UI kits (including shadcn/ui and others) are built on top of Tailwind.
  • Any project needing responsive, dark-mode-ready UI — this very page is one example: resize the window or toggle dark mode above to see it in action.

Traditional CSS vs Tailwind

style.css

.btn {
  padding: 8px 16px;
  background: #2661A8;
  border-radius: 8px;
  color: white;
}

index.html — Tailwind

<button class="px-4 py-2
  bg-hsam-blue rounded-lg
  text-white">
  Button
</button>

Same result — no separate CSS file, no class name to invent.
